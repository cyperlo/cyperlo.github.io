Golang 自定义迭代器

在 Go 语言中，自定义迭代器的实现方式在 Go 1.23 版本之后发生了巨大的变化。随着 iter 标准库的引入，Go 正式支持了原生迭代器（Iterators），这让我们可以像 Python 或 Java 那样优雅地使用 for...range。

怎么理解Golang迭代器？

理解 Go 1.23 引入的这种迭代器设计，关键在于理解 “控制权的反转”。

在传统的迭代中，是你去问容器要数据；而在 Go 的 iter.Seq 中，是容器把数据推给你，而你通过一个回调函数（yield）来决定还要不要。

我们可以把这个过程拆解为三个核心角色：

  1. 三个角色的对话

    为了好理解，我们可以把迭代过程想象成一场对话：

    • 迭代器函数 (The Iterator): 负责跑腿寻找数据的人（比如遍历数组、查询数据库）。
    • Yield 函数 (The Consumer): 你的 for range 循环体，它像是一个“过滤器”。
    • 数据 (The Item): 传递的消息。

  2. 核心逻辑：yield 到底是什么？

    当你写 for v := range MySeq 时，Go 编译器在底层做了这些事：

    1. 它把你的 for 循环体包装成了一个函数，这个函数就是 yield。
    2. yield 接收一个值，并返回一个 bool：
      • 返回 true：表示“我处理完了，请给我下一个值”。
      • 返回 false：表示“我遇到了 break 或者报错了，别再发了！”。

  4. 为什么要这样设计（设计的精妙之处）

    • 内存极其省： 相比于把所有结果存进一个 Slice 再返回，这种方式是“惰性”的。哪怕你有 10 亿个数据，它每次也只处理一个。
    • 组合极其强： 你可以像搭积木一样叠加迭代器（比如先 Filter 再 Map）。
    • 统一了标准： 以前每个人写的迭代器都不一样（有的叫 Next()，有的叫 HasNext()），现在全 Go 语言都统一了。
